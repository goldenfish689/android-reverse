### 五个最值得关注的结论

> **① 需求极度饥渴**：六城市 406 个岗位，且多家公司同时在多城市重复招聘——说明市场根本招不够人，这是稳定的卖方市场。
> 
> **② 薪资中位数远超主流 Android 开发**：全市场 20–40K 占 62%，北京顶薪案例出现了折合月均超 16K 的 200–260K 年薪（16薪），上海/杭州稳定在 35K+ 中位。
> 
> **③ 技术壁垒正在快速拔高**：OLLVM 对抗、VMP 还原、unidbg 模拟执行——三年前这些是"加分项"，现在已经是 5 年以上岗位的硬门槛。越晚入场，追赶成本越高。
> 
> **④ 出海赛道是最新的增量机会**：TikTok、Shopee、WhatsApp 专项逆向岗位集中涌现，中文圈专项资料近乎空白，先发优势窗口期已经打开。
> 
> **⑤ 独立接单比想象中更容易启动**：数据采集公司、云手机厂商、游戏工作室——这三类需求方长期"招不到人"，外包意愿极强。一篇高质量的破防分析文章，就可能带来第一个付费客户。

## 一、行业整体情况分析

### 2.2 技术栈结构总结

#### 主流技术栈组合

**组合 1：通用逆向标配（60%+ 职位）**

```
IDA Pro / JEB  +  Frida  +  Xposed/LSPosed  +  JADX  +  脱壳能力
+  Java 阅读  +  Smali 阅读  +  C/C++ 基础  +  ARM 汇编基础
```


**组合 3：游戏逆向专项（约 20% 职位）**

```
IDA Pro + Frida  +  ARM 汇编  +  C/C++
+  Unity IL2CPP 分析  /  Cocos2d-x Lua 加密还原
+  游戏协议 TCP/UDP 自定义加密还原
+  Wireshark / tcpdump
```

**组合 4：风控对抗专项（约 24% 职位）**

```
Frida + Xposed/LSPosed + Magisk
+ 设备指纹分析（IMEI/OAID/MAC/Android ID）
+ SSL Pinning 绕过  +  反调试对抗
+ Root 检测 / Hook 检测绕过
+ Python 自动化
```

## 三、业务场景分析

### 场景一：游戏逆向

**涉及职位数**：≈82 个（占总量约 20%）

#### 解决的问题

| 子场景 | 解决什么问题 |
| --- | --- |
| 外挂开发（进攻方） | 绕过游戏反作弊，实现自动化操作（脱机服务器、自动刷怪、辅助瞄准） |
| 游戏安全（防守方） | 检测外挂行为，防止游戏协议被模拟，保护 IL2CPP/Lua 代码安全 |
| 游戏协议还原 | 逆向游戏服务端通信协议，复现登录/战斗/道具等完整接口 |
| 游戏资产提取 | 解密 AssetBundle、提取美术/音频等资产 |

#### 技术对抗对象

- 游戏反外挂（TP/BattleEye/ACE）的内存扫描、行为异常检测
- 游戏引擎加密：IL2CPP metadata 加密、Lua bytecode 加密
- 游戏服务端协议加密：自定义 AES/RSA + 私有二进制帧格式
- 模拟器检测、调试检测（阻止 Frida/GDB 附加）

#### 实际用户

| 用户类型 | 说明  |
| --- | --- |
| 游戏工作室 | 批量刷金币、游戏代打、自动化任务 |
| 外挂销售商 | 将外挂商品化售卖给普通玩家 |
| 游戏厂商安全部门 | 从攻击角度测试自身游戏，提升反外挂能力 |

#### 典型需求（来自 JD 原文）

- "负责安卓手游的抓包，破解游戏加密算法，还原核心协议"
- "分析 Unity IL2CPP 或 Cocos2d-x 引擎的运行时机制（对象生命周期、脚本绑定、资源加载等）"
- "了解 unity3d 和 cocos2d 引擎的底层特性和修改方法"
- "开发稳定登录器、脱机协议框架"

## 五、职业路径分析

### 5.1 就业路线（进入公司）

#### 初级（0–1 年，对应约 15% 职位）

**最小技能集（来自 JD 必须项）**：

| 技能  | 具体要求 |
| --- | --- |
| 静态分析 | JADX/Apktool 独立反编译 APK，能读懂 Java/Smali 代码 |
| 动态分析 | Frida 基本使用（执行现成脚本 + 简单修改） |
| 抓包  | Charles/Fiddler 抓 HTTP/HTTPS 明文流量 |
| 基础脱壳 | 能使用 BlackDex/FART 等工具脱出 dex |
| 编程  | Python 写基础自动化；Java 读懂 App 业务逻辑 |
| 系统知识 | 理解 Android APK 结构（Manifest/DEX/SO/资源） |

**推荐学习路径**：

```
第 1–3 个月：
  → 看雪/吾爱破解入门教程 + JADX 拆包实践（选真实 App）
  → 学 Smali 语法，能对简单 Activity 逻辑做静态分析

第 4–6 个月：
  → Frida 官方文档 + Hook 实战（Hook Java 层方法）
  → Charles 抓包 + 手动分析接口（能识别 JSON/Protobuf 差异）
  → 脱壳实战：用 BlackDex/FART 处理简单壳

第 7–12 个月：
  → 开始接触 IDA Pro：反编译 libxxx.so，读懂简单 ARM 函数
  → 完成 1–2 个有始有终的逆向项目，写成文章发布
```

**面试重点**：

1. 能描述完整的 APK 逆向流程
2. 能展示一个真实 App 逆向案例（复杂程度不限）
3. Frida Hook 基本操作（可现场演示）

---

#### 中级（1–5 年，对应约 42% 职位，市场主力段）

**1–3 年必须掌握**：

| 技能  | 具体内容 |
| --- | --- |
| Native 层分析 | IDA Pro 反编译 .so，理解 ARM 汇编，定位加密函数 |
| SSL Pinning 绕过 | Frida + JustTrustMe / objection，处理常见证书绑定 |
| Hook 深度 | 自写 Xposed 模块；Frida 脚本处理参数修改、返回值伪造 |
| 脱壳进阶 | 手工 dump + dex 修复（处理指令抽取壳） |
| 签名算法还原 | 跟踪 Java/Native 签名算法，用 Python 复现 |

**3–5 年在上述基础上增加**：

- ARM 汇编深度阅读：能分析 OLLVM 混淆后的基础逻辑
- C/C++ 动手能力：能写 Frida native 模块或简单 Hook 工具
- 协议逆向：Protobuf 还原、gRPC 接口分析、WebSocket 帧分析
- unidbg 基础使用：能 Call 到 SO 中的签名函数
- 完整对抗周期经验："App 更新 → 对抗失效 → 重新分析"至少经历过 1 轮

---

#### 高级（5 年以上，约 31% 职位）

**差异化能力（来自高级岗 JD 原文）**：

- "熟悉 OLLVM/VMP 等混淆保护技术并具备相关破解经验"
- "具备自研通用脱壳机、指令抽取还原工具、虚拟字节码反编译引擎的工程化能力"
- "有魔改内核，自研 Hook 框架或底层加固对抗实战经验"
- "熟悉 ART 虚拟机底层机制（解释执行、AOT 编译、运行时结构）"
- "具备平台级自动化逆向分析能力建设经验"

**面试重点**：

1. 完整对抗案例：从发现对抗点到给出稳定方案的全过程
2. 代码级展示：自己写的 Hook 框架 / 脱壳工具 / Frida 高级脚本
3. 看雪/吾爱破解上的高质量原创文章
